# Toolport v1.6.1

**Windows auto-update fix when MCP clients hold the gateway open.**

If Cursor or another client spawns `toolport-gateway.exe`, the updater could fail when
NSIS could not overwrite the locked install-dir binary.

---

## What changed

- **Versioned client gateway** — publishes `toolport-gateway-{version}.exe` under
  `%APPDATA%\Roaming\Conduit\bin` and re-points MCP configs away from the install-dir
  copy.
- **Pre-update gateway stop** — before installing an update, Toolport stops only spawned
  gateway processes (not your editor).

**Note:** manual installs and upgrades from 1.6.0 needed the NSIS hook added in
**v1.6.2**.

---

## Upgrade

- **In-app:** click **Toolport v1.x.x** in the sidebar footer.
- **Installer:** [Releases](https://github.com/tsouth89/toolport/releases).

Docker/headless: no desktop change required; `ghcr.io/tsouth89/toolport-gateway:latest`
tracks `main`.
