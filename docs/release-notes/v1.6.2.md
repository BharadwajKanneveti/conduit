# Toolport v1.6.2

**Windows install fix when MCP clients hold the gateway open.**

v1.6.1 stopped spawned gateways before the in-app updater ran, but only once you were
already on 1.6.1. Manual installer downloads and upgrades from 1.6.0 still hit “Error
opening file for writing” on `toolport-gateway.exe` when Cursor or another client had it
running.

This patch adds an NSIS pre-install hook that terminates `toolport-gateway.exe` and
`conduit-gateway.exe` before the installer copies files. Editors can stay open.

---

## Upgrade

- **In-app:** click **Toolport v1.x.x** in the sidebar footer (shows **Update to v…**
  when a release is available).
- **Installer:** [Releases](https://github.com/tsouth89/toolport/releases) — no need to
  close Cursor first.
